# proxmox-vm

Terraform module to create and manage a Proxmox virtual machine using the `bpg/proxmox` provider. This README was generated by AI!

## Requirements

- Terraform 1.5.0 or newer
- Provider `bpg/proxmox` version `~> 0.110.0`

## Usage

```hcl
module "proxmox_vm" {
  source = "../proxmox-vm"

  vm_id          = "100"
  vm_name        = "example-vm"
  vm_description = "Example Proxmox VM"
  node_name      = "proxmox-node"

  bios       = "ovmf"
  cpu_cores  = 2
  memory_mb  = 2048

  tags = {
    environment = "dev"
    owner       = "team"
  }

  disks = [
    {
      datastore_id = "local-lvm"
      size_gb      = 20
      interface    = "scsi"
    }
  ]

  network_devices = [
    {
      bridge      = "vmbr0"
      model       = "virtio"
      vlan_id     = 100
      mac_address = "DE:AD:BE:EF:00:01"
    }
  ]
}
```

## Inputs

- `bios` (string, default: `ovmf`)
  - The BIOS type to use for the VM. Valid values are `seabios` or `ovmf`.

- `disks` (list(object), default: `[]`)
  - A list of disk objects to attach to the VM.
  - Each disk object must include:
    - `datastore_id` (string)
    - `size_gb` (number)
    - `interface` (string)

- `cpu_cores` (number)
  - The number of CPU cores to allocate to the VM.

- `memory_mb` (number)
  - The amount of dedicated memory in megabytes.

- `node_name` (string)
  - The Proxmox node name. Leave blank to use the default node behavior if the source VM is already on the correct node.

- `network_devices` (list(object), default: `[]`)
  - A list of network device objects for the VM.
  - Each network device can include:
    - `bridge` (string)
    - `model` (string)
    - `vlan_id` (number, optional)
    - `mac_address` (string, optional)

- `vm_id` (string)
  - The identifier for the VM.

- `vm_name` (string)
  - The name to assign to the VM.

- `vm_description` (string)
  - A description for the VM.

- `tags` (map(string), default: `{}`)
  - Tags to attach to the Proxmox VM resource.

## Outputs

This module does not currently expose any outputs.

## Notes

- When `bios` is set to `ovmf`, the module automatically configures an EFI disk using `local-lvm`.
- Adjust `datastore_id` values for disks and the EFI disk to match your Proxmox storage configuration.
- Ensure the Proxmox provider is configured properly in your root module.
